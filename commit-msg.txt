feat(ssl): one-click SSL certificate generation (local + Let's Encrypt)

Add a complete SSL certificate management system that lets the admin
generate trusted certificates with one click — no paths to know, no
technical knowledge required.

Backend (Go)
- New file: backend/handlers/ssl.go (~500 lines)
- Four new endpoints (admin-only):
  - GET  /api/v1/admin/ssl/status   — parses current cert with Go's
    crypto/x509 and returns issuer, subject, expiry, days_remaining,
    SANs, key algorithm, serial number, health status
  - POST /api/v1/admin/ssl/generate — one-click generation
    - type: "local"  → openssl self-signed RSA 2048 with proper SANs,
      valid 365 days, written to shared certs volume
    - type: "letsencrypt" → certbot --webroot HTTP-01 challenge via
      shared ACME webroot, real browser-trusted cert valid 90 days,
      staging mode supported for testing without rate limits
  - POST /api/v1/admin/ssl/renew    — re-runs certbot for existing
    Let's Encrypt certs and copies the renewed files
  - POST /api/v1/admin/ssl/install  — paste PEM cert+key for ZeroSSL,
    Cloudflare Origin CA, or any paid CA
- Auto-detects domains from existing settings (main/frontend/backend/admin)
- Validates input: rejects Let's Encrypt for .local/.localhost domains,
  requires email for LE, validates PEM format on install
- Writes 7 new settings keys: ssl_type, ssl_issuer, ssl_generated_at,
  ssl_domains, ssl_email, ssl_expiry, ssl_staging
- After any cert operation, triggers nginx reload via the existing
  shared trigger-file mechanism (no restart needed)
- All operations logged to audit_logs with action SSL_CERT_GENERATED /
  SSL_CERT_RENEWED / SSL_CERT_INSTALLED

Infrastructure (Docker)
- backend/Dockerfile: install openssl, certbot, ca-certificates
- nginx/Dockerfile: create /var/www/certbot directory
- docker-compose.yml:
  - Mount ./certs to backend (rw) so backend can write certs
  - Mount ./certs to nginx (ro) — unchanged
  - New acme-webroot volume shared between backend (certbot writes
    challenges) and nginx (serves /.well-known/acme-challenge/)
  - New letsencrypt volume for /etc/letsencrypt persistence (accounts,
    renewal configs)
  - New env vars: SSL_CERTS_DIR, ACME_WEBROOT

Nginx
- nginx.conf.template: add /.well-known/acme-challenge/ location to
  BOTH the HTTP (port 80) and HTTPS (port 443) server blocks, served
  from /var/www/certbot root. Placed BEFORE the HTTP→HTTPS redirect
  so ACME HTTP-01 verification works even when SSL is enabled.

  - Validates that LE cannot be used for local domains
